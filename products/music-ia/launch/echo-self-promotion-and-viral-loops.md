# Echo Self-Promotion And Viral Loops

## Purpose

This document defines how Echo should spread without expensive paid growth.

## Core Rule

Echo should grow by making users naturally carry Echo outward.

## Echo Native Viral Objects

Echo should create things that are naturally shareable:

### 1. Public artist profiles

Useful because creators want visibility and listeners like to share unusual finds.

### 2. Public libraries

Useful because they show taste and are easy to explain and link.

### 3. Borrowed shelves

Useful because they create social proof and encourage profile exploration.

### 4. Rooms

Useful because they group listening around a mood or intent and feel more alive than genre menus.

### 5. Live moments later

Useful because they create urgency without needing permanent storage.

## Strongest Cheap Viral Loop

1. creator publishes
2. listener saves and organizes
3. listener makes a public library
4. listener shares the library
5. new listener opens Echo through that library
6. some of those listeners become creators

## Product Surfaces That Should Carry Growth

Echo should make these surfaces easy to share:

- artist page
- track page
- library page
- room page later

## Current Built Surfaces

The current product already has a first concrete version of:

- public artist page
- public track page
- public library page
- links from the main listener experience into those public pages
- persistent public library shelves that can keep a stronger identity over time

So this growth layer is no longer only strategic. It now exists in the product.

## What Echo Should Avoid

Avoid:

- spam invites
- aggressive popups
- fake referral pressure
- cluttered growth widgets

## Final Rule

Echo should spread because it is interesting to pass along, not because it begs to be shared.

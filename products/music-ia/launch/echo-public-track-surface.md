# Echo Public Track Surface

## Purpose

This file defines why Echo needs a public page for a single track before the first broader beta.

## Core Rule

The music should be able to travel alone.

Echo should not depend only on:

- the home feed
- the artist page
- the public library

## Why The Track Page Matters

A single-track page gives Echo:

- a cheap share surface
- a simple arrival page
- a cleaner link target for outside discovery
- a lighter viral object than a full profile or full app home

## What It Should Show

- track title
- artist name
- access room
- primary category
- short description
- direct path to the artist
- direct path back into Echo

## What It Should Avoid

- heavy dashboard logic
- admin noise
- deep creator controls
- crowded recommendation blocks

## Current Built State

The current Echo web app now includes:

- a dedicated public track route
- backend support to fetch a single Echo track
- links into the track page from:
  - the main listener feed
  - the public artist page
  - the public library page

## Why This Helps Beta

This gives the first creators and first listeners a clean thing to share.

That matters because Echo is trying to grow through:

- identity
- tracks
- libraries

not by imitating a heavy generic feed.

# Echo Multilayer Category System

## Purpose

This document defines the category system for Echo.

Echo should not depend on a flat music taxonomy like:

- pop
- trap
- lo-fi
- electronic

That is too generic and too weak for a product that wants identity.

## Core Rule

Echo needs more than one category system.

It should use at least four layers.

## Layer 1: Listener-Facing Mood Layer

This is the layer the listener feels first.

It should be simple, visual and low-text.

Suggested logic:

- calm
- drive
- dark
- bright
- dream
- focus
- heat
- drift

This layer is not about technical genre precision.

It is about immediate recognition.

## Layer 2: Sound Identity Layer

This layer is internal-first, but can appear in subtle UI.

It describes:

- vocal
- instrumental
- melodic
- percussive
- cinematic
- ambient
- dense
- minimal

This helps Echo understand what a track feels like beyond genre labels.

## Layer 3: Session Role Layer

This is the non-conventional layer that gives Echo more product identity.

Each track can have a role inside a session:

- opener
- deepener
- contrast
- rebound
- finisher
- wildcard

This is powerful because music is not only about what a song is.

It is also about where it fits in a listening flow.

## Layer 4: Algorithm And Control Layer

This layer stays mostly invisible.

It includes:

- genre tags
- tempo class
- retention class
- save class
- replay class
- skip risk
- report burden
- trust modifiers
- exclusion flags

## Why This Works Better

A normal music app says:

- here is a genre

Echo should say:

- here is a mood
- here is a texture
- here is where this track fits in your session

That creates a discovery product instead of a generic catalog.

## Suggested Visual Rule

Use:

- symbols
- color bands
- motion cues
- card position

Use less:

- dense text labels
- long menus
- hard filters

## Exclusion Layer

Categories should also support rejection.

If the user hides:

- a mood
- a sound identity
- an artist
- a session role

Echo should reduce or eliminate that lane.

This makes the category system useful, not decorative.

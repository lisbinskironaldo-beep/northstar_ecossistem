# Echo Signature Mini Player And Dark Shell Pass

## Purpose

This pass turns the listener shell toward the newly locked Echo premium identity.

The focus is not the whole app at once.

The focus is the first listener impression:

- top mini player
- listener shell background
- bottom navigation
- Aura live strips
- release cards
- listener-home support blocks

## Why This Pass Exists

The product direction is now explicitly:

- dark-cinematic
- glass-like
- music-first
- futuristic without becoming noisy

The mini player is the signature object.

So this pass exists to stop the listener shell from feeling like:

- a leftover light prototype
- a card system with mixed themes
- a collection of blocks without one visual anchor

## Scope

This pass changes:

- `apps/echo-mobile/src/components/mobile-player-dock.tsx`
- `apps/echo-mobile/App.tsx`
- `apps/echo-mobile/src/components/mobile-ui.tsx`
- `apps/echo-mobile/app/index.tsx`
- `apps/echo-mobile/src/components/live-room-card.tsx`
- `apps/echo-mobile/src/components/live-room-panel.tsx`
- `apps/echo-mobile/src/components/social-release-card.tsx`
- `apps/echo-mobile/src/components/home-library-grid.tsx`
- `apps/echo-mobile/src/components/ecosystem-horizon-strip.tsx`

This pass does not yet implement:

- drag-driven expansion physics
- blur-native materials
- shared backend social persistence
- final motion polish

## Visual Goal

The listener shell should feel like:

- black glass
- violet-blue neon restraint
- soft cinematic depth
- one strong object at the top
- brighter readable text

## Signature Object

The mini player should become:

- ultra-slim
- oval / half-moon
- fixed
- dark translucent
- cover-integrated
- dominated by one strong play-pause control

Expanded state should remain:

- fuller
- richer
- secondary to the mini state in identity

## Home Rule

The home should no longer depend on explanatory cards to explain itself.

It should read in this order:

1. signature player
2. Aura
3. new releases
4. library
5. quiet ecosystem horizon

## Completion Criteria

This pass is complete when:

- the top player feels visually unique
- the bottom navigation belongs to the same visual family
- live strips and release cards no longer look like a different theme
- the listener shell reads as one product, not a collection of upgraded pieces

# Echo Live Audio Ephemeral Model

## Purpose

This document defines how live audio could fit Echo later without breaking self-sufficiency.

## Core Rule

Live should be treated as:

- event value
- not storage value

## Why Live Matters

Live audio can help Echo because:

- it creates urgency
- it gives creators a reason to return
- it gives listeners a reason to check in
- it creates moments worth sharing

## Why Live Must Be Careful

Live can also create:

- moderation pressure
- concurrency cost
- support load
- product complexity

## Recommended Model

Use live later as:

- short sessions
- scheduled listening rooms
- creator drop events
- temporary premiere rooms

Archive only when needed.

Default logic should be:

- no permanent archive
- or short retention only

## Final Recommendation

Live audio is a good later layer for Echo.

It is not the next immediate build priority.

# Echo Mobile Cut Glass Visual Language

## Purpose

This document defines the new visual language for Echo mobile.

Important update:

The listener shell is now intentionally moving away from a light pastel reading and toward a darker cinematic-glass premium feel, especially around the player and music-first surfaces.

The goal is to stop the app from feeling generic or overly round.

It should feel:

- cleaner
- sharper
- more premium
- more memorable
- darker and more cinematic in the listener path

Without becoming muddy, noisy or visually cheap.

## Core Shape Rule

The mobile app should no longer lean on soft rounded cards everywhere.

The base shape is now:

- rectangular or square blocks
- light cut corners
- one clipped-corner gesture repeated consistently

In practice:

- low radius
- one corner cut flatter than the others
- strong edge definition

## Accent Rule

Every major block or action surface should carry a slim top detail.

This detail works like a visual signature:

- thin
- horizontal
- calm
- readable

It helps the app feel designed instead of assembled.

## Color Rule

The listener path should use:

- dark translucent glass
- soft blue-violet neon accents
- restrained chrome lines
- opaque light text with real contrast

This creates:

- cinematic depth
- premium softness
- enough air for music artwork and motion
- stronger identity than generic white cards

## Product Rule

The visual language should improve:

- feed cards
- search entry blocks
- library blocks
- Aura live strips
- fixed player
- profile cards

It should not stay limited to one screen.

## Current Built State

The current mobile pass now applies this direction through:

- cut-corner `GlassCard`
- cut-corner action pills and chips
- slim accent bars on major blocks
- lighter pastel-glass surfaces
- sharper bottom navigation blocks
- stronger visual distinction between active and inactive states
- a slim fixed player that now uses a larger cover block, one dominant play control and a separate expansion control

## Fixed Player Rule

The fixed player should not feel like a leftover dark toolbar.

In slim mode it should read as:

- half-moon or oval mini player
- integrated cover art
- title and artist stacked clearly
- one dominant play-pause action
- one separate expansion action
- semi-transparent dark glass with opaque light text

Previous and next belong only to the expanded state.

## Consistency Rule

When a screen still carries:

- high rounded corners
- flat white blocks
- generic chips
- plain return buttons

it should be treated as unfinished.

The same cut-glass discipline must also shape:

- search results
- profile metrics
- plan cards
- utility rails
- back and mode controls

## Ecosystem Rule

This visual language is suitable for reuse later across:

- Echo
- Pulse
- Lumen

But each front should still be able to vary:

- motion
- emphasis
- palette balance
- content density

The shared rule is the shape discipline, not total sameness.

# Echo Creator Workspace Model

## Purpose

This document defines how creator access should work inside Echo.

## Final Rule

Echo should not create a fully separate "artist app."

Echo should also not force every creator to create a second account.

Instead:

- every person starts as a normal account
- creator mode can be activated inside that account
- the creator gets a dedicated workspace
- inside that workspace, the account can manage one or more artist personas

## Why This Fits Echo

In Echo, becoming an artist is easy.

That means the system should assume:

- many listeners may become creators
- many creators may have multiple AI personas
- one person may test multiple styles

So the product should not force:

- one account per artist
- one artist per account

## Proposed Structure

### Layer 1: Listener Identity

Always present:

- private library
- public library choice
- shared library access
- hidden and excluded content
- taste history

### Layer 2: Creator Workspace

Activated only when the listener chooses to create.

Use it for:

- creator setup
- upload
- release state
- artist management
- front targeting later

### Layer 3: Artist Personas

Inside the creator workspace, the account can create:

- artist A
- artist B
- artist C

Each artist persona has:

- own name
- own public profile
- own catalog
- own public library
- own audience signals

## Future Cross-Front Expansion

The same creator workspace should later be able to manage:

- Echo tracks
- Pulse short-form content
- Lumen long-form content

So the creator workspace is an ecosystem layer, not only an Echo feature.

## Monetization Rule For Later

Do not charge for extra artist personas now.

But design the structure so future limits are possible:

- one artist included by default
- extra artist slots later by plan or creator level

## Product Rule

Keep listener mode simple.

Keep creator mode available.

Keep artist personas flexible.

That is the right balance between freedom and control.

## Current Built State

The current Echo web preview now includes:

- a dedicated creator workspace component
- persona switching
- workspace horizon for Echo, Pulse and Lumen
- first-pass active-slot visibility
- first-pass cold and dormant release visibility
- a release map that shows which tracks are active, cold or sleeping
- quick publish actions for repeat releases

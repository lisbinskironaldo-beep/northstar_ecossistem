# Echo Creator Feedback Loop

## Purpose

This document defines how Echo should guide creators after the first upload.

## Core Rule

The creator workspace should not only show status.

It should suggest the next move.

## What The Creator Needs To Understand

At a glance, the creator should know:

- how many tracks are active
- how many are cold
- how many are sleeping
- whether it is smarter to publish, cool down, reactivate or leave the catalog alone

## Product Meaning

The creator should feel:

- I know where my catalog stands
- I know what to do next
- I can publish again without friction

## Current Built State

The current Echo web preview now includes:

- recommended next move text
- focus cards for active, cold and dormant pressure
- release-template reuse
- quick-publish shortcuts
- first-pass per-track advice in the release room
- beta-readiness label and intake checks for the selected persona
- week-one pulse metrics for the selected persona
- operator watchlist for first-wave onboarding
- onboarding state label and path steps inside the workspace

## Future Extension

Later, this feedback loop should also support:

- suggested swap candidate when active slots are full
- suggested dormancy candidate
- suggested reactivation candidate
- weekly creator review
- cross-front creator guidance for Pulse and Lumen

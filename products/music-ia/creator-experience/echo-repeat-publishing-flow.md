# Echo Repeat Publishing Flow

## Purpose

This document defines how Echo should make repeat publishing feel lighter than first-time setup.

## Core Rule

The first upload can ask for more.

The second, third and fourth should ask for less.

## Product Meaning

Once a creator already has:

- a persona
- a room habit
- a category pattern
- a source-tool pattern

the product should reuse that memory instead of forcing full re-entry every time.

## Built Direction

The current Echo web preview now includes:

- clear split between setup and release room
- setup can be skipped when the workspace already has a creator
- quick publish actions
- restore persona name action
- reset-to-main action
- clear-draft action
- reuse-latest-release template action

## Why This Matters

If repeat publishing feels heavy, creators stop.

If repeat publishing feels light, creators keep testing:

- new track
- new lane
- new room
- new persona later

## Future Extension

Later, this flow should also support:

- duplicate a release draft
- swap an active track into dormant state
- scheduled reactivation
- Pulse and Lumen publishing from the same workspace

# Echo Creator Beta Intake Checklist

## Purpose

This checklist defines the minimum bar for moving a creator from `accepted` to `onboarded` in the first Echo beta.

## Intake Rule

Do not treat acceptance as onboarding.

A creator is only really onboarded when the product and operator can see that the creator is ready to participate without constant rescue.

## Minimum Intake Checks

- active persona selected
- basic bio present
- at least `1` active main-room track
- at least `2` tracks in the workspace
- creator understands active versus sleeping catalog logic

## Why This Bar Exists

The first beta should not only test whether people say yes.

It should test whether they can:

- enter
- publish
- stay oriented
- publish again

## Current Product Support

The current Echo web preview now exposes:

- beta readiness label for the selected persona
- readiness checklist items inside the creator workspace
- onboarding state label and path steps
- week-one pulse metrics for the selected persona
- creator goal for the next move
- operator watchlist
- active, cold and dormant release counts
- next-action guidance before the next upload

## Operator Rule

If a creator accepts but fails the intake bar, mark as:

- `accepted`, not yet `onboarded`

Only mark:

- `onboarded`

when the checklist is materially complete.

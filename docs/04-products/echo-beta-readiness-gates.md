# Echo Beta Readiness Gates

## Purpose

This document defines the practical gates that must be true before the first controlled Echo beta is opened to real creators and listeners.

It exists to stop the team from opening beta only because the product is "working enough."

## Core Rule

Echo beta opens only when:

- the first session feels alive
- creators can publish without rescue
- moderation still feels under control
- the operator can understand what is happening quickly

## Gate 1: Product Gate

The product gate is open only if all of these are true:

- the listener can open the feed and immediately see a believable music session
- the player actions work without confusion
- save works
- follow works
- report works
- creator setup works
- upload works
- the listener view and creator view are clearly separated

## Gate 2: Catalog Gate

The catalog gate is open only if all of these are true:

- the opening feed does not feel empty
- there are enough visible tracks to create variety in the first session
- there is a reserve catalog ready to replace weak openers
- at least a few tracks already feel worth saving
- the catalog does not rely only on smoke-test content

## Gate 3: Creator Gate

The creator gate is open only if all of these are true:

- the creator path is short and understandable
- a new creator can create a profile and publish without operator rescue
- the first invited creators fit the Echo idea
- the outreach message is ready
- the cohort tracker is ready to receive real names and responses

## Gate 4: Ops Gate

The ops gate is open only if all of these are true:

- the admin reflects reports and moderation actions correctly
- the operator knows the daily review clock
- issue logging is ready
- rollback rules are written down
- the founder can quickly read what changed, what broke and what needs attention

## Gate 5: Beta Entry Gate

Open the first controlled beta only if the four earlier gates are all open.

Suggested beta shape:

- `10 to 25` creators
- `25 to 75` listeners
- `7 to 14` days
- one small cohort first, not a broad opening

## Hold Triggers

Do not open beta yet if any of these are still happening:

- the feed still feels like a test panel
- the first session still feels empty or repetitive
- creators still need hand-holding to publish
- reports are visible but hard to understand
- the operator cannot tell what to fix first

## Exit Signal From Readiness

Echo is considered beta-ready when the answer to these four questions is "yes":

- would a listener understand the point of Echo in the first minute?
- would a creator know where to start?
- would an operator know what to watch every day?
- would the team feel safe inviting real people this week?

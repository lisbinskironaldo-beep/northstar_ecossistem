# Echo Creator Rescue Boundaries

## Purpose

This file defines how much operator help is acceptable during the first Echo beta.

It exists because too much rescue hides product problems.

## Core Rule

If creators only move because someone keeps explaining every step, the product is not ready enough.

## Acceptable Rescue

Allow:

- one clarification on what Echo is
- one clarification on how to start
- one clarification on active versus sleeping catalog

## Too Much Rescue

Treat this as product failure, not user failure:

- repeated explanation of setup fields
- repeated explanation of upload path
- repeated confusion about where the first track should go
- repeated confusion about what counts as onboarded

## Operator Rule

When rescue repeats:

- log it
- do not hide it
- do not count the creator as healthy

## Daily Read

At end of day, ask:

- who moved with no rescue?
- who moved with one light clarification?
- who still depends on hand-holding?

## Why This Matters

The first wave must tell the truth.

Otherwise Echo will look ready only because the team carried everyone through it.

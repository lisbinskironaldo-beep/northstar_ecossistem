# Echo Accepted Creator Packet

## Purpose

This file is the practical packet to send once a creator has accepted the Echo beta invite.

It should be short, clear and low-noise.

## Recommended Message

```text
Great. Here is the Echo beta path:

1. choose or create your creator persona
2. add a short bio
3. publish one main-room track first
4. if the first flow feels clear, publish a second track
5. tell us what feels confusing, slow or unfair

This is a controlled beta, so the most useful thing you can give us is honest friction.

Echo is discovery-first. There is no payout promise here. The goal is to help strong AI music creators get heard earlier and to learn what still breaks that experience.
```

## What The Creator Should Understand

Keep these points explicit:

- Echo is still controlled beta
- the ask is small
- one first track is enough to start
- second upload matters because it proves repeatability
- active versus sleeping catalog is part of the model

## What Not To Send Yet

Do not overload the creator with:

- ecosystem theory
- Pulse and Lumen details
- future monetization ideas
- long feature lists

## Operator Reminder

After sending this packet:

- status stays `accepted`
- do not mark `onboarded` yet
- wait for real intake completion inside the product

# Echo Creator Week 01 Metrics

## Purpose

This file defines the minimum creator-side metrics that matter in the first controlled Echo beta.

It exists to keep week-one reading simple, operator-friendly and cheap.

## Rule

Week one is not about full analytics.

Week one is about knowing whether a creator persona can:

- enter
- publish
- hold a small opening
- come back
- avoid clogging the catalog

## The Four Metrics

### 1. Abertura

How many active main-room tracks the persona has.

Why it matters:

- without at least one active main-room track, the persona should not be treated as ready
- with two, the opening already feels safer

## 2. Folga ativa

How many active slots are still open inside the `6` active-track cap.

Why it matters:

- zero folga means the next release already requires curation
- healthy folga means the creator can still test one more move without suffocating the catalog

## 3. Publicadas

How many tracks are currently published (`active + cold`).

Why it matters:

- below `3`, week one still has little material to read
- with `3+`, the product can start observing return and selection quality

## 4. Reserva

How many tracks are outside the main opening (`cold + dormant`).

Why it matters:

- zero reserve means there is no fallback or reactivation material
- too much reserve too early can mean clutter, confusion or weak release choice

## Operator Watchlist

The workspace should raise warnings when:

- there is no active main-room track
- there is no second upload yet
- active slots are already full
- cold tracks outnumber active tracks
- intake readiness is still incomplete

## Product Support

The current Echo creator workspace now exposes:

- beta-readiness label
- intake checklist
- week-one pulse metrics
- creator goal for the next move
- operator watchlist

## Why This Matters

This keeps week one realistic.

The product does not need a heavy dashboard yet.

It needs:

- a clean read
- a small number of useful signals
- guidance that helps both the creator and the operator

# Low-Cost Sidecar Growth Model

## Purpose

This document defines whether Northstar should build a cheap secondary growth product before Pulse.

## Core Rule

Do not build a second content front too early.

If Northstar builds a sidecar before Pulse, it must be:

- cheap
- utility-first
- highly shareable
- clearly connected to Echo
- not dependent on heavy media storage

## What A Sidecar Should Not Be

It should not be:

- a second full content platform
- a fake mini-TikTok
- a video-heavy product
- an expensive experimental toy

## Best Current Candidate

The strongest candidate right now is:

- a public library and profile sharing utility

Why it fits:

- cheap to host
- naturally shareable
- pushes traffic back into Echo
- does not require a second heavy content pipeline

## Live Audio As A Side Layer

Live audio is interesting later, but it should not be the first cheap sidecar.

Why:

- it creates urgency
- it can be low-storage if not archived for long

But it still adds:

- moderation pressure
- concurrency concerns
- support complexity

## Final Recommendation

Before Pulse, the best cheap sidecar is:

- utility-like
- share-driven
- media-light
- clearly useful on its own

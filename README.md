# Northstar Ecosystem

`Northstar Ecosystem` e o nome interno provisorio da empresa e da plataforma central.

Este repositorio esta organizado para sustentar a execucao do ecossistema com continuidade entre sessoes. A estrutura foi montada por objetivos, para que o projeto avance com:

- visao de empresa, nao so de app
- sustentabilidade financeira desde o inicio
- expansao global como requisito, nao como ideia futura
- tres frentes separadas para o usuario
- uma operacao central unica por tras

## Nomes provisorios

- Plataforma-mae: `Northstar Ecosystem`
- Frente 1: `Echo` (`Music IA`)
- Frente 2: `Pulse` (`Shorts IA`)
- Frente 3: `Lumen` (`Watch IA`)

Os nomes sao internos e podem ser trocados depois sem quebrar a organizacao.

## Regra de direcao

O usuario vera tres produtos.

A empresa operara um unico sistema central.

## Regra de prioridade

1. Estruturar o core da empresa e da plataforma.
2. Validar `Echo / Music IA`.
3. Abrir `Pulse / Shorts IA`.
4. Abrir `Lumen / Watch IA`.
5. So depois acelerar monetizacao ampla e distribuicao de talentos.

## Preview local atual

- Preview web estavel do Echo: `cmd /c npm run dev:echo` (`http://127.0.0.1:3010`)
- Admin web: `cmd /c npm run dev:admin`
- Command Center web: `cmd /c npm run dev:command`
- API: `cmd /c npm run dev:api` (`http://127.0.0.1:3001`)
- API watch mode secundario: `cmd /c npm run dev:api:watch`
- Echo mobile shell continua no projeto para evolucao futura: `cmd /c npm run dev:echo:mobile`
- Echo mobile em rede local para celular: `cmd /c npm run dev:echo:mobile:lan`
- Echo mobile esta agora alinhado ao `Expo SDK 54`

Observacao:

- o preview oficial de navegador do Echo agora e `apps/echo-web`
- o `apps/echo-mobile` continua como superficie mobile, mas o preview web via Expo foi considerado instavel neste ambiente
- o `apps/echo-mobile` agora usa uma casca mobile mais limpa, com `Inicio`, `Buscar` e `Perfil`, e o modo profissional fica apenas dentro do perfil
- o `apps/echo-mobile` agora tambem tem uma primeira fundacao de audio real para feed preview e player fixo, usando ativos locais de teste servidos pela API
- o `Perfil` mobile agora foi reorganizado em dois caminhos limpos: um pulso de conta para `Ouvinte` e um gate de planos + aceite para `Profissional`
- no `Profissional`, planos pagos agora abrem uma etapa de pagamento antes da troca, e `Studio` ficou separado de `Ao vivo`
- o mobile agora esta migrando para uma linguagem visual de blocos com canto recortado, vidro pastel e detalhe slim no topo das superficies
- a proxima execucao mobile agora esta travada em 4 ondas oficiais: shell e hierarquia, creator path e busca, ritual central (`Estreia em Circulo`) e retencao por descoberta pioneira
- a Wave 1 ja tem uma primeira passada executada: shell mais premium, home reorganizada em `Aura`, `Estreias`, `Novas publicacoes`, `Biblioteca` e `novidades`
- a Wave 1 agora tambem ganhou uma segunda passada de percepcao premium: `Buscar`, `Perfil` e o palco do artista foram puxados para a mesma familia dark-cinematic
- a Wave 1 agora tambem ganhou uma terceira passada de distincao da home: `Aura agora` e `Estreias de hoje` deixaram de parecer a mesma familia de card
- a Wave 1 agora tambem ganhou uma quarta passada de reducao de ruido: menos texto, menos repeticao visual e horizonte do ecossistema mais discreto
- no desenvolvimento local, o `Echo web` usa `apps/echo-web/.env.local` para os IDs demo
- no desenvolvimento local, o comando `dev:echo:mobile:lan` agora atualiza automaticamente o IP da API para a rede atual antes de abrir o Expo Go
- o backend `dev:api` agora roda em modo compilado com watch, porque `tsx watch` se mostrou instavel para DI do Nest neste Windows
